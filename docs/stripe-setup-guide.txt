================================================================
Stripe セットアップ手順書（オーナー様向け）
================================================================

本手順書は、本サービスの決済を担当いただく Stripe アカウント保有者様
（以下、オーナー様）が、Stripe ダッシュボード上で行う作業をまとめた
ものです。すべて Stripe 側の Web 画面のみで完結し、コードを触る必要
はありません。

最後に「6 つの情報」を開発担当者へお渡しいただきます
（末尾「9. 開発担当者へ受け渡す情報」参照）。


----------------------------------------------------------------
0. 事前にご用意いただくもの
----------------------------------------------------------------

以下を手元にご用意のうえ、作業を始めてください。

[事業者情報]
  - 法人名・本店所在地・代表者氏名・生年月日
  - 法人番号（13 桁）／個人事業主の場合は氏名・住所
  - 電話番号・問い合わせ用メールアドレス

[本人確認書類（代表者）]
  - 運転免許証 / マイナンバーカード / 旅券 のいずれか

[入金先の銀行口座情報]
  - 銀行名・支店名・口座種別・口座番号・口座名義（カナ）

[その他]
  - 公開用 Web サイト URL
    本サービスは現在モバイルアプリ（iOS / Android）として提供される
    ため、コーポレートサイトの代わりに「App Store / Google Play のス
    トアページ URL」を Stripe へ登録します。

      App Store URL  : https://apps.apple.com/jp/app/id<App Store Connect で割り当てられた数値 ID>
      Google Play URL: https://play.google.com/store/apps/details?id=com.queue.numbers

    ※ Stripe ダッシュボードの「Web サイト URL」欄が 1 つしか入力でき
       ない場合は、現在ストアで公開されている方（推奨：App Store URL）
       を入力してください。もう一方はサポート用補足としてメールで添える
       か、後日追加します。
    ※ Stripe の本番化審査では「URL を開くと事業内容・料金・特商法表記
       が確認できる」ことが必要です。ストアページに以下が記載されてい
       ることをご確認ください：
         - アプリの説明（事業内容）
         - 価格情報（サブスク料金の説明）
         - 利用規約・プライバシーポリシーへのリンク

  - インボイス制度の登録番号（取得済みの場合）

⚠️ 入力する事業者情報は、Stripe の本人確認審査と銀行口座への入金
   審査に使われます。登記情報・銀行口座名義と完全一致するよう注意
   してください。


----------------------------------------------------------------
1. Stripe アカウントの新規発行
----------------------------------------------------------------

1) ブラウザで https://dashboard.stripe.com/register を開く
2) 業務用メールアドレス・氏名・パスワードを設定して登録
3) 受信したメールから「メールアドレス認証」を完了
4) ログイン後、左上の国選択が「日本」になっていることを確認

💡 この時点ではダッシュボード右上に「テストモード」と表示されています。
   手順 2 までで本番化、手順 3 以降で本番モードに切り替えて作業して
   いきます。


----------------------------------------------------------------
2. アカウントの本番化（Activate Your Account）
----------------------------------------------------------------

ダッシュボード左下「アカウントを有効化」または上部バナー
「アカウントを有効化する」から、案内に沿ってすべて入力します。

【2-1. 事業形態】
  - 法人 / 個人事業主 / 任意団体 などを選択

【2-2. 事業者情報】
  - 法人名（登記どおり）
  - 法人番号
  - 所在地・電話番号
  - 業種（最も近いものを選択：「ソフトウェア」「広告・マーケティング」など）
  - 事業内容の説明（日本語で 1〜2 文程度）
    例：「企業向け求人・インターン掲載のサブスクリプションサービス」
  - Web サイト URL
    → App Store または Google Play のストアページ URL を入力
      （上記「0. 事前にご用意いただくもの」を参照）

      App Store URL  : https://apps.apple.com/jp/app/id<App Store Connect で割り当てられた数値 ID>
      Google Play URL: https://play.google.com/store/apps/details?id=com.queue.numbers

【2-3. 代表者・受益者情報】
  - 代表者氏名・生年月日・住所
  - 役職
  - 持株比率 25% 以上の受益者がいる場合は追加で登録

【2-4. 本人確認書類のアップロード】
  - 上記「0.」の本人確認書類をスマートフォンで撮影してアップロード
  - 表面・裏面の両方を求められた場合は両方アップロード

【2-5. 入金先銀行口座】
  - 銀行名・支店名・口座種別・口座番号・口座名義カナ
  - ⚠️ 名義が登記事業者と一致していること

【2-6. 申請】
  - 入力完了後「送信」を押すと、Stripe 側で審査が始まります
    （通常 1〜3 営業日）
  - 審査中も以降の手順は進められますが、「決済を受け付ける」状態に
    なるのは審査通過後です


----------------------------------------------------------------
3. 公開情報・ブランディングの設定（本番モードで実施）
----------------------------------------------------------------

ダッシュボード右上のモード切替を「本番モード」に切り替えてから
作業します。

【3-1. 公開情報】
  左メニュー「設定」→「ビジネス」→「公開情報」
  - 公開ビジネス名（顧客への請求書・領収書に表示される名称）
  - サポートメールアドレス
  - サポート電話番号
  - サポート URL（特定商取引法表記ページの URL を推奨）
  - 明細書表記
    （クレジットカード明細に表示される 5〜22 文字の英数字）
    例：QUEUE-SUB のような短く分かりやすい文字列

【3-2. ブランディング】
  左メニュー「設定」→「ビジネス」→「ブランディング」
  - ロゴ（推奨 128×128px 以上、PNG）
  - アイコン（正方形）
  - ブランドカラー（任意）

【3-3. 顧客向けメール】
  左メニュー「設定」→「メール」
  - 領収書・請求書の自動送信を「ON」
  - 送信元名は公開ビジネス名と一致させる


----------------------------------------------------------------
4. 商品と価格の作成（本番モードで実施）
----------------------------------------------------------------

本サービスでは以下「3 プラン × 月額のみ = 3 つの価格」を作成します。

  プランコード | 商品名（顧客に見える名前） | 月額（税込）
  -------------+----------------------------+--------------
  standard     | 通常プラン                 | ¥10,999 / 月
  special      | 特別プラン                 |  ¥9,899 / 月
  shokokai     | 商工会プラン               |  ¥1,100 / 月

⚠️ プランコードは Stripe 上には入力しません。開発担当者との照合用に
   「どの Price ID がどのプランか」を記録するためのものです。

【4-1. 商品（Product）の作成】
  左メニュー「商品カタログ」→ 右上「＋ 商品を追加」

  各プランごとに以下を入力：

    商品名              : 上表「商品名」をそのまま入力
    商品の説明（任意）  : 顧客に表示される説明文。空欄でも可
    画像（任意）        : プランのアイコン画像があれば
    料金体系            : 「継続課金（サブスクリプション）」を選択
    金額                : 上表「月額（税込）」を入力
    通貨                : JPY
    請求期間            : 毎月
    税の動作            : 「価格に税を含める」を選択（税込価格のため）

  「商品を保存」をクリックすると、商品ページに価格 ID
  （price_xxxxxxxxxxxxxxxxxxxxxxxxxx）が表示されます。

【4-2. 価格 ID の控え】
  3 つの商品を作成し終えたら、各価格 ID を以下の形で控えてください
  （開発側に提出します）：

    standard 月額 : price_xxxxxxxxxxxxxxxxxxxxxxxxxx
    special  月額 : price_xxxxxxxxxxxxxxxxxxxxxxxxxx
    shokokai 月額 : price_xxxxxxxxxxxxxxxxxxxxxxxxxx

⚠️ テストモードで作った Price ID は本番では使えません。必ず右上が
   「本番モード」の状態で作成したものをお渡しください。


----------------------------------------------------------------
5. カスタマーポータルの設定（本番モードで実施）
----------------------------------------------------------------

顧客（契約企業）が自分でプラン解約・支払方法変更・領収書ダウンロード
を行える画面を有効化します。

左メニュー「設定」→「Billing」→「カスタマーポータル」

  機能の有効化                  : ON
  プランの解約                  : 許可する
                                  （解約タイミングは「請求期間末」）
  解約理由のアンケート          : 任意（収集する場合は ON）
  支払い方法の更新              : 許可する
  請求情報の更新                : 許可する（住所・氏名）
  請求書・領収書の表示          : 許可する
  プランの変更（アップグレード等）: 当面は「許可しない」
                                  （後日要件次第で開放）
  利用規約 URL                  : サービスの利用規約ページ URL
  プライバシーポリシー URL      : サービスのプライバシーポリシー URL

設定後、必ず最下部の「保存」をクリックしてください。


----------------------------------------------------------------
6. Webhook エンドポイントの作成（本番モードで実施）
----------------------------------------------------------------

開発担当者から事前に下記の「Webhook URL」を受け取ってください。

    https://<開発担当者から共有>/functions/v1/stripe-webhook

【6-1. エンドポイントの追加】
  左メニュー「開発者」→「Webhook」→ 右上「＋ エンドポイントを追加」

  - エンドポイント URL : 上で受け取った URL
  - 説明               : 「Queue / Numbers 本番 Webhook」（任意）
  - API バージョン     : 最新版（初期値のままで可）
  - イベントの選択     : 以下 6 つを選択

      checkout.session.completed
      customer.subscription.created
      customer.subscription.updated
      customer.subscription.deleted
      invoice.payment_failed
      invoice.payment_action_required

  「エンドポイントを追加」をクリック。

【6-2. 署名シークレットの控え】
  作成後のエンドポイント詳細画面に「署名シークレット」（whsec_ で
  始まる文字列）が表示されます。

  「クリックして表示」をクリックして表示し、以下の形で控えてください
  （開発側に提出します）：

    STRIPE_WEBHOOK_SECRET : <whsec_ で始まる Webhook 署名シークレット>

⚠️ この値は漏洩時は再発行になります。必ず安全な方法（パスワード
   マネージャー等）で保管してください。


----------------------------------------------------------------
7. API キーの取得（本番モードで実施）
----------------------------------------------------------------

左メニュー「開発者」→「API キー」

【7-1. 秘密鍵（Secret key）の取得】
  「シークレットキー」欄の「キーを表示」をクリックし、sk_live_ で
  始まる文字列を控えてください：

    STRIPE_SECRET_KEY : <sk_live_ で始まる秘密鍵>（例: sk_live_REDACTED）

⚠️ 絶対に外部公開しないでください（メール・Slack 平文・GitHub への
   貼り付け禁止）。開発担当者へお渡しする際は、1Password などの
   機密情報共有ツール経由でお願いします。

【7-2. 公開可能鍵（Publishable key）の取得（念のため）】
  pk_live_ で始まる文字列も控えておきます（現時点では使用しません
  が、将来のために）：

    STRIPE_PUBLISHABLE_KEY : <pk_live_ で始まる公開鍵>（例: pk_live_REDACTED）


----------------------------------------------------------------
8. 税・インボイスの設定（任意・推奨）
----------------------------------------------------------------

【8-1. インボイス制度（適格請求書）】
  左メニュー「設定」→「ビジネス」→「税金 ID」
  - インボイス制度の登録番号（T + 13 桁）を登録すると、Stripe が
    発行する領収書・請求書に自動で記載されます

【8-2. Stripe Tax（自動消費税計算・任意）】
  価格を税込で登録している場合は不要です。本サービスはすべて税込
  価格でプラン設定するため、初期セットアップ時点では Stripe Tax は
  OFF のままで構いません。


----------------------------------------------------------------
9. 開発担当者へ受け渡す情報
----------------------------------------------------------------

すべての作業完了後、以下 6 項目を「暗号化された方法（1Password /
Bitwarden の共有等）」で開発担当者へお渡しください。

  1. STRIPE_SECRET_KEY        : <sk_live_ で始まる秘密鍵>
  2. STRIPE_WEBHOOK_SECRET    : <whsec_ で始まる Webhook 署名シークレット>
  3. STRIPE_PUBLISHABLE_KEY   : <pk_live_ で始まる公開鍵>
  4. standard プラン 月額 Price ID : <price_ で始まる ID>
  5. special  プラン 月額 Price ID : <price_ で始まる ID>
  6. shokokai プラン 月額 Price ID : <price_ で始まる ID>

加えて、以下の情報もご共有ください（後日のサポート対応で必要）：

  - Stripe アカウント ID
    （ダッシュボード「設定」→「アカウント」の acct_xxxxx）
  - 公開ビジネス名・サポート連絡先（手順 3-1 で登録した内容）

11. よくある質問
----------------------------------------------------------------

Q. 審査が通る前にテストモードで作業しても大丈夫ですか？
A. 手順 3〜7 はすべて本番モードで再度行う必要があります。テスト
   モードで作った Product / Price / Webhook は本番では使えません。
   審査完了後に本番モードで作業してください。手順 4〜7 を予習目的で
   テストモードで先行実施するのは問題ありません。

Q. 銀行口座への入金タイミングは？
A. 日本では原則として「週次（毎週木曜締め・翌週水曜入金）」が
   デフォルトです。「設定」→「銀行口座と通貨」から変更可能。

Q. テストカード番号は？
A. 開発側の動作確認で使う 4242 4242 4242 4242 などのテストカードは
   テストモードでのみ有効です。本番モードでは実カードを使用します
   （最初の動作確認は少額プランで実カード課金 → 即返金 が安全）。

Q. キー漏洩に気付いたら？
A. ダッシュボード「開発者」→「API キー」からキーを失効（Roll）
   すれば即座に無効化できます。失効後、新しいキーを開発担当者へ
   再共有してください。Webhook 署名シークレットも同様に再発行可能
   です。
